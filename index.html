SecureTrust – Engenharia de Dados para Detecção de Fraudes Financeiras

Um pipeline com AWS, Python e Governança de Dados

“Fraudes financeiras podem ocorrer em segundos. Nosso desafio: detectar padrões fraudulentos antes que causem prejuízo. Assim nasceu o SecureTrust.”

O Desafio: Fraudes em Transações Digitais

  • 💸 Perdas financeiras recorrentes
  • 🔐 Segurança comprometida
  • 🧯 Resposta lenta e reativa

“A fintech Times vinha sofrendo com perdas causadas por fraudes em abertura de contas e transações eletrônicas. Precisávamos de um processo automatizado, governado e inteligente para prever comportamentos fraudulentos.”

Missão: Criar uma Solução de Dados Preditiva

  • Criar pipeline de ingestão e tratamento via AWS
  • Garantir qualidade, rastreabilidade e governança
  • Expor dados para BI e Machine Learning
  • Treinar modelo preditivo eficaz e interpretável

“A proposta foi construir uma base sólida de dados e aplicar ciência de dados sobre ela para atacar o problema de frente.”

Arquitetura do Pipeline – Da Ingestão ao Insight

“Aqui está a espinha dorsal do projeto... bucket S3 → Glue → PRATA → Athena → modelagem e visualização.”

Camadas do Pipeline de Dados

  • Bronze (RAW): Dados brutos no S3
  • Silver (REF): Dados tratados via Glue (limpeza, enriquecimento)
  • Gold (Serving Layer): Athena com consulta SQL
  • Analytics: Python + Power BI

“Dividimos as camadas conforme boas práticas... facilidade de manutenção.”

Tratamento Automatizado de Dados com Glue

  • 🔁 Substituição de -1 por NaN
  • 🧹 Remoção de duplicatas
  • ❌ Exclusão da coluna intended_balcon_amount
  • 📆 Criação da coluna month_named
  • 🔄 Processamento em batch via Python Shell

“Automatizamos o processo com AWS Glue e pandas...”

Governança de Dados Aplicada

  • ✅ Dados sintéticos e anonimizados
  • 🔐 IAM no S3 e Athena
  • 📜 LGPD (simulado)
  • 🧾 Metadados catalogados (32 colunas, 1M registros)

“Garantimos rastreabilidade, segurança e compliance...”
